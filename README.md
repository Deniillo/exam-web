# exam-web

```text
Я, как инженер кибербезопасности, хочу нивелировать возможный вектор атаки перегрузки сервера (DDoS). Участились случаи когда из-за высокой нагрузки приложение стало деградировать в плане производительности, ввиду чего было решено установить ограничение на возможное кол-во обращений к API в минуту для конкретного IP адреса и/или пользователя.


В случае, если лимит запросов от пользователя превышен, необходимо вернуть ответ HTTP 429 Too many requests и не обрабатывать такой запрос.

Предложите решение (возможно с использованием сторонних библиотек и/или сервисов) и проведите нагрузочное тестирование чтобы продемонстрировать результат.
Для удобства можно воспользоваться bash скриптом с curl запросом в цикле.

```
